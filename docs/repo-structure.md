# Repository Structure

This page explains what each top-level folder is for.

## Top-level intent

- `src/rdf/`: implementation code.
- `scenarios/`: scenario definition files consumed by the loader.
- `gate_policies/`: policy definition files consumed by the release gate.
- `run_history/`: output directories generated by CLI runs.
- `docs/`: documentation for usage and architecture.
- `tests/`: automated checks for loader, runner, gate, and integration smoke flow.

## Code module responsibilities

- `adapters/`: interface and implementations for sending turns to an assistant backend.
- `scenarios/`: file parsing and validation for scenario inputs.
- `judging/`: conversion of transcript/response data into criterion-level results.
- `execution/`: concurrency, retry, timeout, and run orchestration logic.
- `gates/`: policy rule evaluation and final decision generation.
- `reporting/`: Markdown summary generation.
- `storage/`: JSON/Markdown artifact persistence to filesystem.
- `cli/`: argument parsing and end-to-end wiring for one run.

## Documentation responsibilities

- `system-overview.md`: end-to-end flow.
- `contracts.md`: shared interfaces and models.
- `run-artifacts.md`: run output files and how to read them.
- `scenario-and-policy-authoring.md`: how to author scenario/policy files.
- `examples/sample-report.md`: sample summary format.
